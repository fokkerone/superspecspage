# How It Works

SuperSpecs organizes AI-driven development into four phases. Each phase solves a distinct failure mode.

## Phase 0 — Setup

Run once per project. Revisit when the stack changes.

Define the project's tech stack through a guided questionnaire. Get concrete recommendations for specialist skills, ecosystem libraries, and a production-readiness checklist.

**Command:** `/superspecs:techstack`

## Phase 1 — Plan

Before any code exists, the plan must fit in a fresh 200k-token context window.

1. **Discuss** (`/superspecs:discuss`) — capture decisions before planning
2. **Spec** (`/superspecs:spec`) — write OpenSpec-style requirements (SHALL statements + GIVEN/WHEN/THEN)
3. **Grill** (`/superspecs:grill`) — stress-test the spec; blocks execution until READY

## Phase 2 — Execute

Parallel execution. Each subagent gets a clean context. No shared state between tasks.

1. **Pick Spec** — validate the next spec, confirm it fits a context window
2. **Branch** — one git branch per spec
3. **Subagent** — RED → GREEN → REFACTOR per task; code review after each task
4. **Code Review** — critical issues block all progress

## Phase 3 — Verify

Full test suite. Every spec scenario verified by a test. Then distill everything into the wiki.

**Commands:** `/superspecs:check-tests` → `/superspecs:wiki`

## Phase 4 — Ship

PR. Changelog. Archive. Start the next cycle.

**Command:** `/superspecs:ship`
